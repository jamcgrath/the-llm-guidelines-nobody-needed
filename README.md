# the-llm-guidelines-nobody-needed

A collection of front-end coding guidelines I wrote in late 2024 for use as LLM context in our AI coding assistant setup. We were using [Aider](https://aider.chat/) at the time and wanted to give the LLM a consistent style to follow.

Most of these turned out to be unnecessary as AI tooling improved. We still use a few guidelines on our team, but the ones in this repo are the ones that didn't survive.

## What's in here

- **semantic-html-guidelines.md** - Guidelines for writing semantic HTML
- **patterns.md** -  Clean Code Pattern Guidelines
- **markdown-optimization.md** - How to write token-efficient Markdown for LLM context
- **css-responsive-guidelines.md** - Responsive CSS conventions
- **css-guidelines-full.md** - A comprehensive CSS style guide

## Why share these

I wrote a blog post about the experience of creating these: [The AI Coding Guidelines Nobody Needed (And What They Taught Me)](https://jamesmcgrath.net/blog/ai-coding-guidelines-nobody-needed)

Even though these guidelines are mostly retired, the process of making them taught me a lot about providing context to AI coding assistants and thinking about token efficiency. Maybe they're useful as a reference or a starting point for someone else going through the same process.
